Конспект проблем (актуально на 2026-04-20)

1) gRPC-клиент в интерактивном режиме обрабатывает ввод построчно
- Реально: 1 строка = 1 SQL-запрос.
- Следствие: многострочные команды (например, CREATE TABLE с переносами) ломаются.
- Симптомы: "ERROR: Expected name", затем "ERROR: Unexpected token" на следующих строках.
- Обходной путь: писать команду в одну строку или запускать через script.sql.

2) Интерактивный gRPC-клиент некорректен для нескольких команд в одной строке
- Пример: "CREATE DATABASE test; USE test;" в одном вводе.
- Возможны частичные/неожиданные результаты из-за отсутствия нормального разделения в interactive-пути.
- Рекомендация: одна SQL-команда за ввод.

3) Выход из интерактива работает только с ';'
- Поддержано: "exit;" и "quit;".
- Без ';' ("exit") уходит в SQL-парсер и даёт ошибку.

4) Не реализованы WHERE BETWEEN и WHERE LIKE
- Текущий парсер WHERE ожидает только сравнение вида lhs op rhs.
- BETWEEN/LIKE приводят к ошибкам парсинга.

5) Не поддержаны составные условия WHERE (AND/OR, скобки)
- Даже при наличии токенов ключевых слов, логические выражения в WHERE не исполняются.

6) Не поддержан синтаксис database.table
- Обращение вида "SELECT ... FROM test.users" не работает.

7) Персистентность каталога неполная между перезапусками
- Файлы таблиц/индексов (.tbl/.idx) создаются на диске.
- Но каталог БД в памяти не восстанавливается при новом запуске процесса.
- Следствие: после перезапуска "USE test;" может не находить существующую на диске БД.

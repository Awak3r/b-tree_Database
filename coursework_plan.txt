План выполнения курсовой по СУБД (вариант: классическое B‑tree)

0) Определиться с границами
1. Вариант индекса: классическое B‑tree.
2. Реализуется базовая часть задания (пункт 0), без дополнительных подпунктов 1–12.
3. Стандарт C++: C++17 или C++20.
4. Режимы: интерактивный и пакетный.

1) Архитектура проекта
1. CLI и режимы запуска.
2. Лексер + парсер + AST.
3. Семантическая валидация.
4. Ядро СУБД (каталоги, БД, таблицы).
5. Хранение на диске (файлы/страницы/записи).
6. Индексы (B‑tree на диске).
7. Исполнение запросов.
8. JSON‑вывод.

2) Хранение данных
1. Папка системы и папки БД.
2. Файл каталога БД (схемы таблиц).
3. Файл таблицы данных.
4. Файлы индексов для INDEXED.
5. RID (ссылка на запись).
6. Free‑list для свободных слотов.

3) Индекс B‑tree
1. Ключи: int и string.
2. Значение: RID.
3. Операции: find/insert/delete/range.
4. Узлы в страницах.
5. Заголовок индекса: root_id, порядок.

4) Парсер и семантика
1. CREATE/DROP/USE.
2. CREATE/DROP TABLE.
3. INSERT/UPDATE/DELETE/SELECT.
4. WHERE: == != < > <= >=, BETWEEN, LIKE.
5. Проверка типов, NOT_NULL, INDEXED‑уникальность.

5) Исполнение запросов
1. CREATE/DROP DB.
2. CREATE/DROP TABLE.
3. INSERT.
4. UPDATE.
5. DELETE.
6. SELECT.
7. Использование индекса при WHERE.

6) Вывод
1. SELECT возвращает JSON‑массив объектов.
2. NULL → null.

7) Надёжность
1. Минимум: корректное сохранение на диск.
2. Желательно: журналирование (если успеешь).

8) Тестирование и защита
1. Юнит‑тесты индекса.
2. Скрипты интеграционных запросов.
3. Демонстрация сохранения после перезапуска.


Граф выполнения по дням (сжатые сроки)

День 1
1. Создать каркас проекта, структуру папок, CMake.
2. Заготовить классы: Database, Table, Record, Catalog.
3. Определить формат записи и RID.
4. Сделать чтение/запись файла таблицы (минимум).

День 2
1. Реализовать базовый B‑tree in‑memory.
2. Операции: insert/find/erase/range.
3. Покрыть тестами B‑tree.
4. Подготовить интерфейс индексного слоя (ключ → RID).

День 3
1. Перевести B‑tree на дисковые страницы (page manager).
2. Реализовать загрузку/сохранение узлов.
3. Сделать заголовок индекса (root_id, order).
4. Проверить корректность на тестах.

День 4
1. Реализовать лексер (токены) и парсер (AST).
2. Команды CREATE/DROP/USE.
3. CREATE/DROP TABLE.
4. Семантика для схемы и типов.

День 5
1. Реализовать INSERT.
2. Обновлять индекс для INDEXED полей.
3. Проверка NOT_NULL/INDEXED.
4. Сохранение данных на диск.

День 6
1. Реализовать SELECT + WHERE.
2. Подключить индекс для поиска.
3. JSON‑вывод результата.
4. Тесты выборок.

День 7
1. Реализовать UPDATE.
2. Обновление индексов при смене ключа.
3. Семантические проверки.

День 8
1. Реализовать DELETE.
2. Удаление из индекса.
3. Проверить целостность.

День 9
1. Пакетный режим (script.txt).
2. Проверка ошибок и стабильности.
3. Интеграционные тесты.

День 10
1. Финальная шлифовка.
2. Скрипт демонстрации.
3. Подготовка к защите: ответы, схема данных, объяснение B‑tree.
4. Черновик пояснительной записки.


Рекомендация
Использовать свою реализацию B‑tree: в задании нет явного разрешения на внешние библиотеки.
